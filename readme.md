# Gamification Admin Platform

## 1. Purpose of the Platform

The **Gamification Admin Platform** is a management dashboard for creating, scheduling, monitoring, and analysing customer quests in a banking digital channel.

A **quest** is a customer challenge or campaign. For example, a customer may be encouraged to:

- Move their salary to the bank.
- Save money in a specific account.
- Use mobile banking services.
- Pay bills on time.
- Spend using a bank card.
- Donate to charity.
- Complete safe banking behaviours.

The platform helps the business team manage these quests from one place and understand whether they are improving customer engagement, revenue, savings behaviour, responsible banking, and risk reduction.

---

## 2. Main Business Goal

The platform is designed to support **prize-linked banking campaigns**.

This means customers complete specific banking actions, earn points or XP, and may become eligible for rewards or prizes. At the same time, the bank can track whether these campaigns are helping business objectives such as:

- Increasing CASA balances.
- Increasing digital engagement.
- Encouraging card spending.
- Reducing risky customer behaviour.
- Supporting social responsibility initiatives.

---

## 3. Who Will Use This Platform?

This platform is mainly intended for business and operations users such as:

- Retail banking teams.
- Digital banking teams.
- Marketing teams.
- Product owners.
- Campaign managers.
- Customer engagement teams.
- Risk and compliance teams.
- Management users who want to monitor campaign performance.

The platform does not require the user to understand programming. It is built as an admin dashboard with forms, tables, calendars, charts, and filters.

---

## 4. High-Level User Journey

A business user can use the platform in the following way:

1. Create a new customer quest.
2. Define the target customer segment.
3. Set the start and end dates.
4. Assign the quest to a business category.
5. Enter the expected target users, completion rate, reward points, revenue, and prize budget.
6. View the quest in the campaign calendar.
7. Monitor all quests in the digital channel table.
8. Track top-performing customers in the ranking table.
9. Review charts and KPIs to understand performance.
10. Export the quest list as a CSV file.

---

## 5. Dashboard Overview

The top section of the platform acts as a **command center**. It provides a quick management summary of the overall gamification program.

It displays four key performance indicators:

| KPI | Meaning |
|---|---|
| Active / Scheduled Quests | Number of campaigns currently live or planned. |
| Customers Doing Quests | Estimated number of customers participating in quests. |
| Forecast Revenue | Expected revenue from all quests. |
| Forecast ROI | Expected return after deducting prize budget. |

This allows management to quickly understand whether the gamification program is active, reaching customers, and producing business value.

---

## 6. Quest Categories

Each quest belongs to one strategic category. These categories help the bank organise campaigns by business objective.

| Category | Business Meaning | Example Quest |
|---|---|---|
| Improve CASA | Encourage deposits, savings, and salary movement. | Move salary account to the bank. |
| Increase Engagement | Encourage customers to use digital banking more often. | Pay bills using the mobile app. |
| Encourage Spending | Encourage customers to use cards or accounts for purchases. | Buy coffee using the bank card. |
| Risk Reduction and Good Behavior | Encourage safer and more responsible customer behaviour. | Pay credit card bills on time. |
| Social Responsibility | Encourage charity, sustainability, and community actions. | Donate to a verified charity. |

The platform shows summary cards for these categories, including the number of quests, participating customers, and estimated ROI.

---

## 7. Quest Creation Form

The **Quest Forge Form** is the form used to create or edit a quest.

The user enters the following information:

| Field | Description |
|---|---|
| Quest Title | Name of the campaign or challenge. |
| Category | The strategic objective of the quest. |
| Status | Current stage of the quest. |
| Start Date | When the quest begins. |
| End Date | When the quest ends. |
| Customer Segment | Target group of customers. |
| Target Users | Number of customers expected to be targeted. |
| Completion % | Expected percentage of customers who will complete the quest. |
| XP Reward | Points or reward value given to customers. |
| Forecast Revenue | Expected business revenue from the quest. |
| Prize Budget | Cost allocated for rewards or prizes. |
| Customer Action | What the customer must do to complete the quest. |

The form also checks that the start date is not after the end date.

---

## 8. Quest Statuses

Each quest can have one of the following statuses:

| Status | Meaning |
|---|---|
| Draft | The quest is still being prepared. |
| Scheduled | The quest is planned for a future date. |
| Live | The quest is currently active. |
| Paused | The quest is temporarily stopped. |
| Completed | The quest has finished. |

These statuses help the team manage the campaign lifecycle from planning to completion.

---

## 9. Campaign Calendar

The **Campaign Calendar** gives a visual view of when quests are running.

Users can:

- View quests by month or by week.
- Move to previous or next periods.
- Jump to today.
- Filter quests by category.
- Filter quests by status.
- Click a quest to view or edit details.
- Use the plus button on a date to quickly create a quest for that day.

The calendar also shows whether a day is lightly loaded, balanced, or heavily loaded with campaigns. This helps avoid too many campaigns running at the same time.

---

## 10. Calendar Summary

The calendar includes a period summary showing:

| Metric | Meaning |
|---|---|
| Quests in Period | Number of quests running in the selected period. |
| New Launches | Number of quests starting in the selected period. |
| Customers Doing | Estimated customers participating. |
| Forecast Revenue | Expected revenue from quests in that period. |

This helps campaign managers understand the workload and expected impact for a selected month or week.

---

## 11. Upcoming Launch Agenda

The **Upcoming Launch Agenda** shows the next quests scheduled to start in the selected calendar period.

For each upcoming quest, the platform displays:

- Quest title.
- Category.
- Start date.
- Target customer segment.
- Number of targeted customers.

This helps teams prepare for upcoming campaign launches.

---

## 12. Digital Channel Quest Table

The **Digital Channel Quest Table** is a full list of all quests that may be published to customer-facing channels such as:

- Mobile banking.
- Internet banking.
- Branch tablet journeys.
- Call center journeys.

The table allows users to:

- Search for quests.
- Filter by category.
- Filter by status.
- Sort columns.
- Control the number of rows shown.
- View quest details.
- Edit quests.
- Delete quests.
- Export quests as a CSV file.

The table includes important business details such as schedule, targeted customers, forecast revenue, prize budget, ROI, and status.

---

## 13. Customer Ranking

The **Top 10 Customer Ranking** shows the customers with the highest quest activity.

The ranking is based on:

- Completed quests.
- XP points earned.
- Completion percentage.
- Estimated business impact.
- Prize tier.

Prize tiers include:

| Prize Tier | Meaning |
|---|---|
| Diamond | Highest-value customers in the leaderboard. |
| Platinum | Very strong performers. |
| Gold | Good performers. |
| Silver | Active participants. |

This leaderboard can support reward eligibility, customer recognition, and campaign engagement tracking.

---

## 14. Analytics and Charts

The platform includes several charts to help management understand performance.

### Revenue & Engagement Forecast

Shows expected revenue and estimated participating customers by month.

This helps management understand whether future campaigns are expected to increase business value and engagement.

### Quest Mix

Shows how quests are distributed across strategic categories.

This helps ensure the campaign portfolio is balanced and not focused on only one objective.

### Customer Funnel

Shows the movement from:

1. Targeted customers.
2. Participating customers.
3. Completed quest customers.

This helps identify whether campaigns are successfully converting targeted customers into active participants.

### Strategic KPI Scorecard

Shows performance across strategic objectives such as CASA, engagement, spending, risk, and social responsibility.

This gives a management-level view of which objectives are performing strongly and which may need more attention.

## 17. Important Calculations Used by the Platform

The platform calculates several business metrics automatically.

| Calculation | Explanation |
|---|---|
| Customers Doing Quests | Target users multiplied by expected completion percentage. |
| Forecast ROI | Forecast revenue minus prize budget, divided by prize budget. |
| Category ROI | Revenue and budget performance by quest category. |
| Completion Forecast | Expected percentage of targeted customers completing the quest. |
| Customer Impact | Estimated business uplift created by top customers. |

These calculations help business users compare campaigns and understand expected value.

---

## 18. Data Storage Behaviour

The current version stores quest data in the browser using local storage.

This means:

- The dashboard can remember quests created by the user on the same browser.
- Reloading the page does not immediately remove the saved quests.
- The sample data can be reloaded using the reload button.

For a production banking environment, this would normally be connected to a secure backend system and database.

---

## 19. Limitations to Consider Before Production Use

The current HTML version is a front-end prototype. Before using it as a real banking platform, the following areas should be reviewed:

- User authentication and access control.
- Secure database storage.
- Approval workflow for launching quests.
- Audit trail for edits and deletions.
- Integration with mobile banking and internet banking channels.
- Integration with customer data and transaction systems.
- Compliance review for prize-linked campaigns.
- Data privacy controls.
- Maker-checker approval process.
- Role-based permissions for business, risk, compliance, and admin users.

This platform is a **banking gamification admin dashboard**.

# To start the server
Start Docker Desktop, then:

```
# Terminal 1 — DynamoDB Local
docker compose up -d dynamodb-local

# Terminal 2 — seed data
cd backend && node seed.js

# Terminal 3 — start server
cd backend && npm run dev
```
